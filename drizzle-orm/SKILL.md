# Drizzle ORM

This skill provides comprehensive guidance for working with Drizzle ORM in TypeScript projects. Drizzle is a lightweight, type-safe SQL-first ORM that maps directly to SQL concepts. It produces zero overhead at runtime, generates no extra abstraction layers, and gives full control over queries while maintaining complete type safety.

## 1. Installation and Setup

### PostgreSQL

```bash
npm install drizzle-orm postgres
npm install -D drizzle-kit
```

### MySQL

```bash
npm install drizzle-orm mysql2
npm install -D drizzle-kit
```

### SQLite (Node.js with better-sqlite3)

```bash
npm install drizzle-orm better-sqlite3
npm install -D drizzle-kit @types/better-sqlite3
```

### SQLite (Bun)

```bash
bun add drizzle-orm
bun add -D drizzle-kit
```

### Recommended Project Structure

```
src/
├── db/
│   ├── index.ts          # Database connection and drizzle instance
│   ├── schema/
│   │   ├── index.ts      # Re-exports all schema modules
│   │   ├── users.ts      # User-related tables and relations
│   │   ├── posts.ts      # Post-related tables and relations
│   │   └── enums.ts      # Shared enums (PostgreSQL)
│   └── migrations/       # Generated by drizzle-kit
├── services/
│   ├── user.service.ts
│   └── post.service.ts
└── drizzle.config.ts     # At project root
```

## 3. Database Connection

### PostgreSQL with `postgres` (postgres.js)

```typescript
// src/db/index.ts
import { drizzle } from "drizzle-orm/postgres-js";
import postgres from "postgres";
import * as schema from "./schema";

const connectionString = process.env.DATABASE_URL!;

// For query purposes (connection pool)
const client = postgres(connectionString, {
  max: 10,
  idle_timeout: 20,
  connect_timeout: 10,
});

export const db = drizzle(client, { schema, logger: true });

// For migrations only (single connection, then close)
export function createMigrationClient() {
  const migrationClient = postgres(connectionString, { max: 1 });
  return drizzle(migrationClient);
}
```

### PostgreSQL with `node-postgres` (pg)

```typescript
import { drizzle } from "drizzle-orm/node-postgres";
import { Pool } from "pg";
import * as schema from "./schema";

const pool = new Pool({
  connectionString: process.env.DATABASE_URL!,
  max: 20,
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000,
});

export const db = drizzle(pool, { schema, logger: true });
```

### MySQL with mysql2

```typescript
import { drizzle } from "drizzle-orm/mysql2";
import mysql from "mysql2/promise";
import * as schema from "./schema";

const pool = mysql.createPool({
  uri: process.env.DATABASE_URL!,
  waitForConnections: true,
  connectionLimit: 10,
});

export const db = drizzle(pool, { schema, mode: "default" });
```

### SQLite with better-sqlite3

```typescript
import { drizzle } from "drizzle-orm/better-sqlite3";
import Database from "better-sqlite3";
import * as schema from "./schema";

const sqlite = new Database("local.db");
sqlite.pragma("journal_mode = WAL");
sqlite.pragma("foreign_keys = ON");

export const db = drizzle(sqlite, { schema });
```

### Singleton Pattern (Next.js / Hot Reload Safe)

```typescript
// src/db/index.ts
import { drizzle } from "drizzle-orm/postgres-js";
import postgres from "postgres";
import * as schema from "./schema";

const globalForDb = globalThis as unknown as {
  client: ReturnType<typeof postgres> | undefined;
};

const client =
  globalForDb.client ?? postgres(process.env.DATABASE_URL!, { max: 10 });

if (process.env.NODE_ENV !== "production") {
  globalForDb.client = client;
}

export const db = drizzle(client, { schema });
```

## 4. Schema Definition

### PostgreSQL Tables

```typescript
// src/db/schema/users.ts
import {
  pgTable,
  pgEnum,
  uuid,
  varchar,
  text,
  boolean,
  timestamp,
  integer,
  index,
  uniqueIndex,
} from "drizzle-orm/pg-core";
import { relations } from "drizzle-orm";
import { posts } from "./posts";

export const roleEnum = pgEnum("role", ["user", "editor", "admin"]);

export const users = pgTable(
  "users",
  {
    id: uuid("id").defaultRandom().primaryKey(),
    email: varchar("email", { length: 255 }).notNull().unique(),
    name: varchar("name", { length: 255 }).notNull(),
    password: varchar("password", { length: 255 }).notNull(),
    role: roleEnum("role").default("user").notNull(),
    active: boolean("active").default(true).notNull(),
    createdAt: timestamp("created_at", { withTimezone: true })
      .defaultNow()
      .notNull(),
    updatedAt: timestamp("updated_at", { withTimezone: true })
      .defaultNow()
      .notNull()
      .$onUpdate(() => new Date()),
  },
  (table) => [
    uniqueIndex("users_email_idx").on(table.email),
    index("users_created_at_idx").on(table.createdAt),
  ]
);

export const profiles = pgTable("profiles", {
  id: uuid("id").defaultRandom().primaryKey(),
  bio: text("bio"),
  avatar: varchar("avatar", { length: 512 }),
  userId: uuid("user_id")
    .notNull()
    .unique()
    .references(() => users.id, { onDelete: "cascade" }),
});
```

```typescript
// src/db/schema/posts.ts
import {
  pgTable,
  uuid,
  varchar,
  text,
  boolean,
  integer,
  timestamp,
  jsonb,
  index,
  primaryKey,
} from "drizzle-orm/pg-core";
import { relations } from "drizzle-orm";
import { users } from "./users";

export const posts = pgTable(
  "posts",
  {
    id: uuid("id").defaultRandom().primaryKey(),
    title: varchar("title", { length: 255 }).notNull(),
    slug: varchar("slug", { length: 255 }).notNull().unique(),
    content: text("content").notNull(),
    published: boolean("published").default(false).notNull(),
    views: integer("views").default(0).notNull(),
    metadata: jsonb("metadata").$type<{
      readTime?: number;
      featured?: boolean;
    }>(),
    authorId: uuid("author_id")
      .notNull()
      .references(() => users.id, { onDelete: "cascade" }),
    categoryId: uuid("category_id").references(() => categories.id, {
      onDelete: "set null",
    }),
    publishedAt: timestamp("published_at", { withTimezone: true }),
    createdAt: timestamp("created_at", { withTimezone: true })
      .defaultNow()
      .notNull(),
    updatedAt: timestamp("updated_at", { withTimezone: true })
      .defaultNow()
      .notNull()
      .$onUpdate(() => new Date()),
  },
  (table) => [
    index("posts_author_id_idx").on(table.authorId),
    index("posts_slug_idx").on(table.slug),
    index("posts_published_idx").on(table.published, table.createdAt),
  ]
);

export const categories = pgTable("categories", {
  id: uuid("id").defaultRandom().primaryKey(),
  name: varchar("name", { length: 100 }).notNull().unique(),
  slug: varchar("slug", { length: 100 }).notNull().unique(),
  parentId: uuid("parent_id"),
});

export const tags = pgTable("tags", {
  id: uuid("id").defaultRandom().primaryKey(),
  name: varchar("name", { length: 50 }).notNull().unique(),
});

// Many-to-many junction table
export const postsToTags = pgTable(
  "posts_to_tags",
  {
    postId: uuid("post_id")
      .notNull()
      .references(() => posts.id, { onDelete: "cascade" }),
    tagId: uuid("tag_id")
      .notNull()
      .references(() => tags.id, { onDelete: "cascade" }),
  },
  (table) => [
    {
      pk: primaryKey({ columns: [table.postId, table.tagId] }),
    },
  ]
);

export const comments = pgTable(
  "comments",
  {
    id: uuid("id").defaultRandom().primaryKey(),
    body: text("body").notNull(),
    postId: uuid("post_id")
      .notNull()
      .references(() => posts.id, { onDelete: "cascade" }),
    authorId: uuid("author_id")
      .notNull()
      .references(() => users.id, { onDelete: "cascade" }),
    parentId: uuid("parent_id"),
    createdAt: timestamp("created_at", { withTimezone: true })
      .defaultNow()
      .notNull(),
  },
  (table) => [
    index("comments_post_id_idx").on(table.postId),
    index("comments_author_id_idx").on(table.authorId),
  ]
);
```

## 5. Relations

Drizzle relations are defined separately from table definitions and are used exclusively by the relational query API (`db.query.*`). They do not alter the database schema or add foreign keys.

```typescript
// src/db/schema/relations.ts
import { relations } from "drizzle-orm";
import { users, profiles } from "./users";
import { posts, categories, tags, postsToTags, comments } from "./posts";

// User relations
export const usersRelations = relations(users, ({ one, many }) => ({
  profile: one(profiles, {
    fields: [users.id],
    references: [profiles.userId],
  }),
  posts: many(posts),
  comments: many(comments),
}));

export const profilesRelations = relations(profiles, ({ one }) => ({
  user: one(users, {
    fields: [profiles.userId],
    references: [users.id],
  }),
}));

// Post relations
export const postsRelations = relations(posts, ({ one, many }) => ({
  author: one(users, {
    fields: [posts.authorId],
    references: [users.id],
  }),
  category: one(categories, {
    fields: [posts.categoryId],
    references: [categories.id],
  }),
  comments: many(comments),
  postsToTags: many(postsToTags),
}));

// Category self-relation
export const categoriesRelations = relations(categories, ({ one, many }) => ({
  parent: one(categories, {
    fields: [categories.parentId],
    references: [categories.id],
    relationName: "categoryTree",
  }),
  children: many(categories, { relationName: "categoryTree" }),
  posts: many(posts),
}));

// Many-to-many through junction table
export const postsToTagsRelations = relations(postsToTags, ({ one }) => ({
  post: one(posts, {
    fields: [postsToTags.postId],
    references: [posts.id],
  }),
  tag: one(tags, {
    fields: [postsToTags.tagId],
    references: [tags.id],
  }),
}));

export const tagsRelations = relations(tags, ({ many }) => ({
  postsToTags: many(postsToTags),
}));

// Comment self-relation (threaded replies)
export const commentsRelations = relations(comments, ({ one, many }) => ({
  post: one(posts, {
    fields: [comments.postId],
    references: [posts.id],
  }),
  author: one(users, {
    fields: [comments.authorId],
    references: [users.id],
  }),
  parent: one(comments, {
    fields: [comments.parentId],
    references: [comments.id],
    relationName: "commentThread",
  }),
  replies: many(comments, { relationName: "commentThread" }),
}));
```

## 6. Migrations with drizzle-kit

### CLI Commands

```bash
# Generate migration SQL from schema changes
npx drizzle-kit generate

# Apply migrations to the database
npx drizzle-kit migrate

# Push schema directly (no migration files -- useful for prototyping)
npx drizzle-kit push

# Pull existing database schema into Drizzle schema files
npx drizzle-kit pull

# Open Drizzle Studio (visual database browser)
npx drizzle-kit studio

# Check for schema drift without generating
npx drizzle-kit check

# Drop a migration file
npx drizzle-kit drop
```

### Programmatic Migration

```typescript
// src/db/migrate.ts
import { migrate } from "drizzle-orm/postgres-js/migrator";
import { drizzle } from "drizzle-orm/postgres-js";
import postgres from "postgres";

async function runMigrations() {
  const connection = postgres(process.env.DATABASE_URL!, { max: 1 });
  const db = drizzle(connection);

  console.log("Running migrations...");
  await migrate(db, { migrationsFolder: "./src/db/migrations" });
  console.log("Migrations complete.");

  await connection.end();
}

runMigrations().catch((err) => {
  console.error("Migration failed:", err);
  process.exit(1);
});
```

### Migration Workflow

```bash
# 1. Modify schema files (add columns, tables, etc.)
# 2. Generate migration
npx drizzle-kit generate

# 3. Review generated SQL in src/db/migrations/
# 4. Apply migration
npx drizzle-kit migrate

# Alternative: push directly for rapid prototyping (no migration files)
npx drizzle-kit push
```

### Custom Migration Edits

After generating, edit the SQL file in the migrations folder to add data transformations:

```sql
-- src/db/migrations/0003_add_full_name.sql
ALTER TABLE "users" ADD COLUMN "full_name" varchar(255);
UPDATE "users" SET "full_name" = "name" WHERE "name" IS NOT NULL;
ALTER TABLE "users" ALTER COLUMN "full_name" SET NOT NULL;
```

## 7. CRUD Operations

### Insert (Create)

```typescript
import { db } from "../db";
import { users, posts, postsToTags, tags } from "../db/schema";

// Single insert
const [newUser] = await db
  .insert(users)
  .values({
    email: "alice@example.com",
    name: "Alice",
    password: hashedPassword,
    role: "admin",
  })
  .returning();

// Insert with specific returning columns
const [created] = await db
  .insert(users)
  .values({
    email: "bob@example.com",
    name: "Bob",
    password: hashedPassword,
  })
  .returning({ id: users.id, email: users.email });

// Batch insert
const newUsers = await db
  .insert(users)
  .values([
    { email: "user1@example.com", name: "User 1", password: hashed },
    { email: "user2@example.com", name: "User 2", password: hashed },
    { email: "user3@example.com", name: "User 3", password: hashed },
  ])
  .returning();

// Insert with conflict handling (upsert)
const [upserted] = await db
  .insert(users)
  .values({
    email: "alice@example.com",
    name: "Alice Updated",
    password: hashedPassword,
  })
  .onConflictDoUpdate({
    target: users.email,
    set: {
      name: sql`excluded.name`,
      updatedAt: new Date(),
    },
  })
  .returning();

// Insert and ignore conflicts
await db
  .insert(users)
  .values({ email: "alice@example.com", name: "Alice", password: hashed })
  .onConflictDoNothing({ target: users.email });
```

### Select (Read)

```typescript
import { eq, ne, gt, gte, lt, lte, like, ilike, and, or, not,
         inArray, between, isNull, isNotNull, sql, desc, asc, count } from "drizzle-orm";

// Select all columns
const allUsers = await db.select().from(users);

// Select specific columns
const userEmails = await db
  .select({
    id: users.id,
    email: users.email,
    name: users.name,
  })
  .from(users);

// Where clauses
const activeAdmins = await db
  .select()
  .from(users)
  .where(and(eq(users.active, true), eq(users.role, "admin")));

// Complex filtering
const filteredPosts = await db
  .select()
  .from(posts)
  .where(
    and(
      eq(posts.published, true),
      gte(posts.publishedAt, new Date("2025-01-01")),
      or(
        ilike(posts.title, "%typescript%"),
        ilike(posts.content, "%typescript%")
      ),
      not(isNull(posts.categoryId))
    )
  );

// Ordering
const sortedPosts = await db
  .select()
  .from(posts)
  .where(eq(posts.published, true))
  .orderBy(desc(posts.publishedAt), asc(posts.title));

// Limit and offset (pagination)
const page = 1;
const pageSize = 20;
const paginatedPosts = await db
  .select()
  .from(posts)
  .where(eq(posts.published, true))
  .orderBy(desc(posts.createdAt))
  .limit(pageSize)
  .offset((page - 1) * pageSize);

// Count
const [{ total }] = await db
  .select({ total: count() })
  .from(users)
  .where(eq(users.active, true));

// Distinct
const uniqueRoles = await db
  .selectDistinct({ role: users.role })
  .from(users);

// Find first (single record)
const user = await db
  .select()
  .from(users)
  .where(eq(users.email, "alice@example.com"))
  .limit(1)
  .then((rows) => rows[0] ?? null);
```

### Relational Query API (db.query)

The relational API provides a Prisma-like interface powered by the `relations()` definitions:

```typescript
// Find many with nested relations
const usersWithPosts = await db.query.users.findMany({
  columns: {
    id: true,
    email: true,
    name: true,
    password: false, // Exclude sensitive fields
  },
  with: {
    profile: true,
    posts: {
      where: eq(posts.published, true),
      orderBy: [desc(posts.createdAt)],
      limit: 5,
      with: {
        comments: {
          with: {
            author: {
              columns: { id: true, name: true },
            },
          },
          limit: 10,
        },
        postsToTags: {
          with: { tag: true },
        },
      },
    },
  },
  where: eq(users.active, true),
  orderBy: [asc(users.name)],
  limit: 50,
});

// Find first
const userByEmail = await db.query.users.findFirst({
  where: eq(users.email, "alice@example.com"),
  with: {
    profile: true,
  },
});

// Nested filtering with many-to-many
const postsWithTag = await db.query.posts.findMany({
  where: eq(posts.published, true),
  with: {
    author: { columns: { id: true, name: true } },
    postsToTags: {
      with: { tag: true },
    },
  },
});
```

### Update

```typescript
// Update single record
const [updatedUser] = await db
  .update(users)
  .set({ role: "editor" })
  .where(eq(users.id, userId))
  .returning();

// Increment a value
const [updatedPost] = await db
  .update(posts)
  .set({ views: sql`${posts.views} + 1` })
  .where(eq(posts.id, postId))
  .returning();

// Conditional update with multiple fields
const [updated] = await db
  .update(posts)
  .set({
    published: true,
    publishedAt: new Date(),
    updatedAt: new Date(),
  })
  .where(and(eq(posts.id, postId), eq(posts.published, false)))
  .returning();

// Bulk update
await db
  .update(users)
  .set({ active: false })
  .where(
    and(
      eq(users.active, true),
      lt(users.createdAt, new Date("2024-01-01"))
    )
  );
```

### Delete

```typescript
// Delete single record
const [deleted] = await db
  .delete(users)
  .where(eq(users.id, userId))
  .returning();

// Bulk delete
const deletedCount = await db
  .delete(sessions)
  .where(lt(sessions.expiresAt, new Date()));

// Delete with subquery
await db
  .delete(comments)
  .where(
    inArray(
      comments.postId,
      db.select({ id: posts.id }).from(posts).where(eq(posts.authorId, userId))
    )
  );
```

## 8. Joins

Drizzle supports SQL-native joins with full type inference.

### Inner Join

```typescript
const postsWithAuthors = await db
  .select({
    postId: posts.id,
    postTitle: posts.title,
    authorName: users.name,
    authorEmail: users.email,
```

### Transaction with Rollback

```typescript
async function transferCredits(
  fromId: string,
  toId: string,
  amount: number
) {
  return db.transaction(async (tx) => {
    const [sender] = await tx
      .select({ id: users.id, credits: users.credits })
      .from(users)
      .where(eq(users.id, fromId))
      .for("update"); // SELECT ... FOR UPDATE (row-level lock)

    if (!sender || sender.credits < amount) {
      tx.rollback();
      // Alternatively: throw new Error("Insufficient credits");
    }

    await tx
      .update(users)
      .set({ credits: sql`${users.credits} - ${amount}` })
      .where(eq(users.id, fromId));

    await tx
      .update(users)
      .set({ credits: sql`${users.credits} + ${amount}` })
      .where(eq(users.id, toId));

    const [txRecord] = await tx
      .insert(transactions)
      .values({
        fromUserId: fromId,
        toUserId: toId,
        amount,
        type: "credit_transfer",
      })
      .returning();

    return txRecord;
  });
}
```

### Cursor-Based Pagination

```typescript
async function getCursorPaginatedPosts(
  cursor?: string,
  take: number = 20
) {
  const conditions = [eq(posts.published, true)];

  if (cursor) {
    const cursorPost = await db
      .select({ createdAt: posts.createdAt })
      .from(posts)
      .where(eq(posts.id, cursor))
      .limit(1)
      .then((rows) => rows[0]);

    if (cursorPost) {
      conditions.push(lt(posts.createdAt, cursorPost.createdAt));
    }
  }

  const results = await db
    .select()
    .from(posts)
    .where(and(...conditions))
    .orderBy(desc(posts.createdAt))
    .limit(take + 1);

  const hasNextPage = results.length > take;
  const data = hasNextPage ? results.slice(0, -1) : results;

  return {
    data,
    nextCursor: hasNextPage ? data[data.length - 1].id : null,
    hasNextPage,
  };
}
```

## 15. Dynamic Query Building

```typescript
interface PostFilters {
  search?: string;
  authorId?: string;
  categoryId?: string;
  published?: boolean;
  tags?: string[];
  dateFrom?: Date;
  dateTo?: Date;
  sortBy?: "date" | "views" | "title";
  sortOrder?: "asc" | "desc";
  page?: number;
  pageSize?: number;
}

async function queryPosts(filters: PostFilters) {
  const conditions = [];

  if (filters.published !== undefined) {
    conditions.push(eq(posts.published, filters.published));
  }
  if (filters.authorId) {
    conditions.push(eq(posts.authorId, filters.authorId));
  }
  if (filters.categoryId) {
    conditions.push(eq(posts.categoryId, filters.categoryId));
  }
  if (filters.search) {
    conditions.push(
      or(
        ilike(posts.title, `%${filters.search}%`),
        ilike(posts.content, `%${filters.search}%`)
      )
    );
  }
  if (filters.dateFrom) {
    conditions.push(gte(posts.createdAt, filters.dateFrom));
  }
  if (filters.dateTo) {
    conditions.push(lte(posts.createdAt, filters.dateTo));
  }
  if (filters.tags?.length) {
    conditions.push(
      inArray(
        posts.id,
        db
          .select({ postId: postsToTags.postId })
          .from(postsToTags)
          .innerJoin(tags, eq(postsToTags.tagId, tags.id))
          .where(inArray(tags.name, filters.tags))
      )
    );
  }

  const where = conditions.length > 0 ? and(...conditions) : undefined;

  const sortColumn = {
    date: posts.createdAt,
    views: posts.views,
    title: posts.title,
  }[filters.sortBy ?? "date"];

  const orderDir =
    filters.sortOrder === "asc" ? asc(sortColumn) : desc(sortColumn);

  const page = filters.page ?? 1;
  const pageSize = filters.pageSize ?? 20;

  const [data, [{ total }]] = await Promise.all([
    db
      .select()
      .from(posts)
      .where(where)
      .orderBy(orderDir)
      .limit(pageSize)
      .offset((page - 1) * pageSize),
    db.select({ total: count() }).from(posts).where(where),
  ]);

  return {
    data,
    meta: {
      total,
      page,
      pageSize,
      totalPages: Math.ceil(total / pageSize),
    },
  };
}
```

## 19. Type Inference

Drizzle provides comprehensive type inference from the schema definition.

```typescript
import { InferSelectModel, InferInsertModel } from "drizzle-orm";
import { users, posts } from "./schema";

// Infer types from schema
type User = InferSelectModel<typeof users>;
type NewUser = InferInsertModel<typeof users>;
type Post = InferSelectModel<typeof posts>;
type NewPost = InferInsertModel<typeof posts>;

// Partial types for updates
type UserUpdate = Partial<Omit<NewUser, "id" | "createdAt">>;

// Custom select types
type UserPublic = Pick<User, "id" | "email" | "name" | "role">;

// Type-safe service layer
class UserService {
  constructor(private db: typeof db) {}

  async create(data: NewUser): Promise<User> {
    const [user] = await this.db.insert(users).values(data).returning();
    return user;
  }

  async findById(id: string): Promise<User | null> {
    const result = await this.db
      .select()
      .from(users)
      .where(eq(users.id, id))
      .limit(1);
    return result[0] ?? null;
  }

  async update(id: string, data: UserUpdate): Promise<User> {
    const [updated] = await this.db
      .update(users)
      .set(data)
      .where(eq(users.id, id))
      .returning();
    return updated;
  }
}

// Type from relational query result
type UserWithPosts = Awaited<
  ReturnType<typeof db.query.users.findFirst<{
    with: { posts: true; profile: true };
  }>>
>;
```

## 20. Drizzle vs Prisma -- When to Choose Drizzle

### Choose Drizzle When

- **SQL-first mindset**: Comfortable with SQL and want queries that map directly to SQL constructs
- **Performance-critical**: Need zero overhead with no runtime query engine or binary dependency
- **Serverless/edge environments**: Drizzle has no native binary, making it ideal for Cloudflare Workers, Vercel Edge, Deno Deploy
- **Bundle size matters**: Drizzle is ~50KB vs Prisma's ~15MB engine
- **Schema in TypeScript**: Prefer defining schema in code rather than a DSL (`.prisma` files)
- **Multiple databases**: Need to support PostgreSQL, MySQL, and SQLite with the same codebase patterns
- **Granular control**: Want full control over joins, subqueries, CTEs, and raw SQL without leaving the type system
- **Existing database**: Use `drizzle-kit pull` to generate schema from an existing database

### Choose Prisma When

- **Rapid prototyping**: Prisma's schema DSL and auto-generated client reduce boilerplate for simple CRUD
- **Team familiarity**: Team members are not comfortable writing SQL-like code
- **GUI tooling**: Prisma Studio provides a visual data browser
- **Prisma ecosystem**: Need Prisma Accelerate (connection pooling) or Prisma Pulse (real-time subscriptions)
- **Complex nested writes**: Prisma's nested create/connect/disconnect API is more ergonomic for deep relation mutations in a single call

### Key Differences

| Feature | Drizzle | Prisma |
| --- | --- | --- |
| Schema definition | TypeScript files | `.prisma` DSL |
| Query style | SQL-like builder + relational API | Prisma Client API |
| Type safety | Inferred from schema (compile-time) | Generated client (code-gen) |
| Runtime dependency | None (pure JS/TS) | Rust query engine binary |
| Bundle size | ~50KB | ~15MB (includes engine) |
| Migration | `drizzle-kit generate/migrate/push` | `prisma migrate dev/deploy` |
| Raw SQL | `sql` tagged template (typed) | `$queryRaw` / `$executeRaw` |
| Edge/serverless | Native support | Requires Prisma Accelerate |
| Joins | Explicit SQL joins | Implicit via `include`/`select` |
| Transactions | `db.transaction()` | `$transaction()` (batched + interactive) |
| Relation API | `db.query.*` (Prisma-like) | Built-in with `include`/`select` |
